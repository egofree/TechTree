# Static timing analysis (STA)

> **Type**: material | **Tier**: important | **Domains**: vlsi-scaling

Analyzes every path in the design to determine maximum delay (setup check) and minimum delay (hold check). Unlike simulation, STA checks all paths simultaneously — it is exhaustive. Reports timing violations with specific paths, slack values, and suggested fixes. STA is the primary timing signoff tool — a design must pass STA across all process corners before tapeout.
